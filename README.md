## 🎯 About SwiftCV

**SwiftCV** is a 100% free, production-ready ATS resume builder built specifically for **Indian students and freshers**. We help students create resumes that actually get past the Applicant Tracking System (ATS) bots and reach human recruiters.

```
🎓 Fill resume   →   📊 Get ATS score   →   🤖 AI rewrites bullets
       ↓                     ↓                        ↓
🎯 Paste job desc  →  ✅ Score hits 80+  →  📥 Download free PDF
```

> **Core Promise:** A student from a Tier-3 college with no career cell should have the same resume quality as someone from IIT. SwiftCV is their career counselor.

## 📊 ATS Score System

SwiftCV's ATS score is calculated across **5 pillars** — 20 points each = **100 total**:

<div align="center">

| Pillar | Max Score | What it checks |
|:---:|:---:|---|
| 📋 **Format** | 20 pts | Email, phone, LinkedIn, GitHub present & valid |
| 📝 **Content** | 20 pts | Summary, sections filled, word count adequate |
| 🔍 **Keywords** | 20 pts | Skills listed, action verbs used, role match |
| 💪 **Impact** | 20 pts | Quantified bullets, metrics, numbers used |
| ✅ **Completeness** | 20 pts | Projects, certifications, achievements filled |

</div>

<br/>

```
Score Legend:
🔴  0  – 40   →  Needs Major Work     (most Canva/Word resumes land here)
🟡  41 – 70   →  Getting There        (missing keywords or quantification)
🟢  71 – 100  →  Ready to Apply  🎉   (target this before submitting)
```

## 🚀 Getting Started

### Prerequisites

```bash
node --version   # v18.0.0 or higher
npm --version    # v8.0.0 or higher
git --version    # any recent version
```

### Installation

**1️⃣ Clone the repo**
```bash
git clone https://github.com/yourusername/swiftcv.git
cd swiftcv
```

**2️⃣ Install dependencies**
```bash
npm install
```

**3️⃣ Set up environment variables**
```bash
cp .env.example .env.local
# Fill in your API keys (see Environment Variables below)
```

**4️⃣ Set up Supabase tables**

Run this in your Supabase SQL editor:

```sql
-- Profiles table
CREATE TABLE profiles (
  id UUID REFERENCES auth.users(id) PRIMARY KEY,
  full_name TEXT,
  email TEXT,
  target_role TEXT,
  college TEXT,
  grad_year TEXT,
  ai_rewrites_used INTEGER DEFAULT 0,
  ai_rewrites_date DATE,
  created_at TIMESTAMPTZ DEFAULT now()
);
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users own profile" ON profiles FOR ALL USING (auth.uid() = id);

-- Resumes table
CREATE TABLE resumes (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  title TEXT DEFAULT 'My Resume',
  data JSONB NOT NULL DEFAULT '{}',
  ats_score INTEGER DEFAULT 0,
  share_token TEXT UNIQUE DEFAULT gen_random_uuid()::text,
  is_public BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
ALTER TABLE resumes ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users own resumes" ON resumes FOR ALL USING (auth.uid() = user_id);

-- Resume views tracking
CREATE TABLE resume_views (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  resume_id UUID REFERENCES resumes(id) ON DELETE CASCADE,
  viewed_at TIMESTAMPTZ DEFAULT now(),
  ip_hash TEXT
);
```

**5️⃣ Run the dev server**
```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) 🎉

## 🔑 Environment Variables

```env
# ─── Supabase ──────────────────────────────────────────────────
```

## 🚢 Deploying to Vercel

**1.** Push to GitHub → Go to [vercel.com](https://vercel.com) → New Project → Import repo

**2.** Add all env variables from `.env.local` in the Vercel dashboard

**3.** Update Supabase Auth settings:
- Site URL: `https://your-domain.vercel.app`
- Redirect URL: `https://your-domain.vercel.app/auth/callback`

**4.** Deploy 🚀

```bash
# Or via CLI:
npm install -g vercel
vercel --prod
```
